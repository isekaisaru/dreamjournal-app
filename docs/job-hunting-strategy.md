## 戦略の核心

**「未経験です」ではなく「準実務者として伝える」**

```
❌ 「未経験ですが頑張ります」
✅ 「実務経験はないが、本番運用・課金・CI/CD まで回している。加えて管理職経験がある」
```

「未経験です」と言った瞬間に選考が「育成コスト計算モード」に入る。
「本番運用・課金・CI/CD まで回している」から入ると「即戦力候補モード」で話が始まる。

- 「未経験歓迎」の文言（ポテンシャル採用・カジュアル面談可で探す）
- 最初から最新技術ど真ん中

- 最初から完璧な条件一致

## 面接で語れる「技術の設計意図」3本柱

これを各1分で声に出して言えるようにする。

### 1. Stripe Subscription の設計意図
- なぜ月額500円にしたか
- Webhook の冪等性をどう担保したか（`stripe_event_id` でデデュプリケーション）
- `client_reference_id` でユーザー解決する設計の意図

### 2. JWT + HttpOnly Cookie を選んだ理由
- localStorage に保存するとXSSリスクがある
- HttpOnly Cookie ならJSからアクセス不可でセキュア
- クロスドメイン（Vercel × Render）での Cookie 設定の工夫

### 3. Next.js + Rails API 分離にした理由
- フロントとバックの責務を明確に分ける
- Vercel（フロント）× Render（バック）でそれぞれ最適化できる
- チーム開発を想定した場合の拡張性

> **おまけ（N+1問題）**
> Railsエンジニアの面接で9割の確率で出る。
> ユメログの `DreamFilterQuery` で `includes(:emotions)` した事例で説明できる。

## 転職フェーズ設計

### フェーズ1（今） - 接点づくり
Wantedly・Green でカジュアル面談の数を増やす。
「応募数」より「会話数」が大事。
話すと伝わるタイプなので、まず場数を踏む。

### フェーズ2（話す練習後） - エージェント活用
非公開求人へのアクセス + 現実ラインの把握。
カジュアル面談3〜5本こなしてから登録する（自己紹介が固まった状態で）。

### フェーズ3（6〜12ヶ月） - 1社目
「理想100点」より「実務に入れる70点」を取る。
絶対条件だけは守る。最初の1社で全部取ろうとしない。
