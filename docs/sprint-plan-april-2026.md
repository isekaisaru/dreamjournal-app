## 🎯 転職活動の進め方

### 基本方針
- 応募は**条件が合う場合のみ**
- 無理に件数を追わない
- まず**接点づくり**を進める
- **開発成果を転職に変換**する

### 接点づくりの具体例
- Findy の「いいかも」
- Wantedly / Green の「気になる」
- 企業フォロー / 採用ページ確認
- カジュアル面談打診 / 求人保存

## ⚠️ リスク管理
| リスク | 対応策 |
|---|---|
| 条件に合う求人が少ない | 応募数より接点づくりを重視し、QA / 品質 / 改善寄りポジションも見る |
| APIコストが不安 | まずレート制限を終えてからユーザー導線強化に進む |
| 開発が止まりそう | DALL·E 3 / 月次サマリーは最小版で止めてよい。**草維持、README更新、テスト修正は死守** |
| モルペウスが単調 | 4月は最適化と配置改善を優先し、差分画像は余力または5月に回す |
