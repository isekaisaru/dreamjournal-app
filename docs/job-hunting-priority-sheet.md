# 転職条件の優先順位シート
> 作成日：2026-04-08
> 目的：求人を見るたびに気持ちが揺れすぎないよう、判断の軸を先に決める

---

## 0. このシートの目的

私は転職活動で、
「何でもいいから受かる会社」を探すのではなく、
「家族・働き方・将来の市場価値」を守りながら前に進める会社を選ぶ。

---

## 1. 絶対に守る条件（1つでも大きく外れるなら原則見送り）

### 働き方
- [ ] フルリモート確定、または家族生活を壊さない働き方
- [ ] 山形在住でも現実的に働ける（週2日以上の出社は不可）
- [ ] 残業が多すぎない
- [ ] 家族との時間を守れる

### 仕事内容
- [ ] Webアプリ開発に関われる
- [ ] 今後の市場価値につながる経験が積める
- [ ] ただの保守だけで終わりにくい
- [ ] 技術や開発経験が職務経歴書に残る

### 自分の気持ち
- [ ] この会社で働く自分を想像したとき、強い違和感がない
- [ ] 現職から逃げるだけではなく、前向きな理由で入れる

---

## 2. できれば満たしたい条件

### 技術
- [ ] Rails / Next.js / TypeScript に近い
- [ ] Git / PR / テスト / CI/CD の文化がある
- [ ] Docker / AWS / GCP などに触れられる
- [ ] AI活用やモダンな開発文化がある（Claude Code / Cursor等）

### 会社の種類
- [ ] 自社開発 / プロダクト開発
- [ ] ユーザーに価値を届けるサービスに関われる
- [ ] エンドユーザーの反応が見えやすい
- [ ] SES色が強すぎない

### 働く環境
- [ ] フレックス制度がある
- [ ] 学習支援やオンボーディングがある
- [ ] チーム開発経験を積める
- [ ] 年齢や経歴に偏見が少ない

---

## 3. 最初の1社では妥協してもいい条件

- [ ] 初年度年収が理想より少し低い
- [ ] 会社の知名度がそこまで高くない
- [ ] 最初の肩書きが華やかではない
- [ ] 最初の会社が完全な自社開発ではない
- [ ] 最初の案件が100%理想の技術スタックではない

---

## 4. 絶対に避けたい条件（1つでも該当するなら見送り）

- フルリモートと書いてあるが実態は不明
- 山形在住だと実質難しい（週2日以上の出社が発生する）
- 残業が多い / 家族時間が壊れそう
- 希望と違う案件に流されやすい（SESの案件ガチャ）
- 技術スタックが今後につながらない
- ただの人月商売で成長イメージが持てない
- 面談しても仕事内容が曖昧
- 「未経験歓迎」と言いつつ実態は経験者前提
- 自社開発と書いてあるが、実際はほぼSES
- 強い違和感があるのに、不安だけで受けようとしている

---

## 5. SESを検討するときの特別ルール

SESを完全否定しない。ただし、通過点として使える場合のみ検討する。

### SESでも受けてよい条件（全部満たすこと）
- [ ] フルリモートが契約レベルで現実的に確約できる
- [ ] Rails / Next.js など今後につながる案件に入れる確率を数字で確認できた
- [ ] 待機時の給与保証がある
- [ ] 希望外案件を断りやすい
- [ ] レビュー・テスト・PR文化がある現場に入れる可能性がある
- [ ] 1〜2年後に自社開発へ移るための経験が積める

### SESなら避ける条件
- フルリモートが「案件次第」で実質ほぼ出社になる
- Java / インフラ / 保守など希望外に流されそう
- 案件の中身が入社前に確定しない
- 実務経験を積める保証が弱い

---

## 6. 今の自分の第一優先順位

```
① フルリモート確定（山形在住・家族生活を守る絶対条件）
② Webアプリ開発経験が職務経歴書に残ること
③ 個人開発（ユメログ）の強みが評価される会社
```

---

## 7. 今の自分の現実的戦略

```
本命：
タイミーのような自社開発 × フルリモート確定 × Rails/Next.js企業
（週1〜2社ペースで探し続ける）

現実的候補：
フルリモートが契約レベルで確約できるSES
ただし「Rails/Next.js案件に入れる確率」を面談で数字として確認してから判断

情報収集候補：
MotherComputer
（個人開発の市場価値を単価で可視化するために面談する。入社前提なし）
```

---

## 8. 求人を見るときの判断フロー

```
① 「フルリモートは確定か？」
     NO → 即見送り

② 「Webアプリ開発経験が積めるか？」
     NO → 即見送り

③ 「1〜2年後に市場価値は上がるか？」
     NO → 見送り

全部YES → 次のステップへ（詳細確認・カジュアル面談）
```

---

## 9. 今の私の暫定結論

- 本命は自社開発・プロダクト開発企業への直接挑戦
- SESは「フルリモート確約 + Rails/Next.js案件保証」が取れた場合のみ踏み台として検討
- フルリモートは一切妥協しない（生活が壊れるから）
- 年収は最初の1社では最優先にしない
- 43歳・未経験・地方在住という条件は不利だが、ユメログの本番運用実績で正面から勝負する
- 焦って条件の悪い会社に入らない
