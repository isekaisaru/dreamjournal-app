# ユメログ マネタイズ設計メモ

作成日：2026-03-30
ステータス：設計検討中（未実装）

---

## 現状

- 現在の課金：Stripe Checkout による任意の寄付（¥500 固定）
- 課金ユーザー数：少数（寄付は「おまけ」として位置づけ）
- API コスト：OpenAI（分析・音声・画像生成）が主な変動費

---

## 問題意識

ユーザーが増えたとき、API コストが無制限にかかる構造になっている。
現状は Rack::Attack でレート制限をかけているが、「無料で使える範囲」と
「有料で使える範囲」の境界線が設計されていない。

---

## マネタイズの方向性

### 方針：**API 利用量に連動したプラン制限**

「夢を記録する」という基本機能は無料で提供し、
AI を使う機能（分析・音声・画像生成）を利用回数で制限する。

---

## 機能差の設計案

| 機能 | 無料プラン | 有料プラン（月額 ¥300〜¥500 想定） |
|---|---|---|
| 夢の記録・閲覧・編集・削除 | ✅ 無制限 | ✅ 無制限 |
| 感情タグ付け | ✅ 無制限 | ✅ 無制限 |
| 月次サマリー表示 | ✅ 閲覧可 | ✅ 閲覧可 |
| AI 夢分析（gpt-4o-mini） | 月 10 回 | 月 100 回（実質無制限） |
| 音声入力（Whisper） | 月 5 回 | 月 50 回 |
| 夢のビジュアル生成（DALL-E 3） | 月 3 回 | 月 30 回 |
| 月次 AI サマリーメッセージ | ❌ なし | ✅ あり |

> 無料枠の数値は Rack::Attack の現行制限を参考に暫定設定。
> 有料プランは Stripe Subscription（月額）で実装予定。

## 技術的に必要な実装

### Phase 1（プランの土台）
- [ ] `users` テーブルに `plan` カラム追加（`free` / `premium`）
- [ ] Stripe Subscription の導入（現在は one-time payment のみ）
- [ ] サブスクリプション成功 Webhook で `plan` を `premium` に更新
- [ ] キャンセル Webhook で `plan` を `free` に戻す

### Phase 2（利用量カウント）
- [ ] `api_usage_logs` テーブルの追加（user_id, feature, used_at）
- [ ] 各 API アクション（分析・音声・画像）で月次カウントを確認
- [ ] 無料枠超過時は `402 Payment Required` を返す
- [ ] フロントで「今月あと○回使えます」表示

### Phase 3（UX）
- [ ] 有料プランへの誘導画面（枠超過時のモーダル）
- [ ] サブスクリプション管理画面（Stripe Customer Portal）
- [ ] 支払い状況の確認ページ

## 価格設計の考え方

**¥300〜¥500 / 月** を想定している理由：

- ターゲットが家族・子育て世帯なので高額は避ける
- Spotify（¥980）・Note（¥500〜）などと比較して「試しやすい」範囲
- OpenAI コストは月額 ¥100〜¥200 程度で収まる想定（有料ユーザー1人あたり）
- 競合は「夢日記アプリ」ではなく「育児記録アプリ」が近い
  - Babilio: 無料〜¥480/月
  - パパっと育児: 無料（寄付型）

---

## 寄付と月額サブスクの使い分け

| 形態 | 適した場面 | 現状 |
|---|---|---|
| 寄付（¥500 一回） | 「応援したい」気持ちに応える | ✅ 実装済み |
| 月額サブスク | 継続的な API 利用のコスト回収 | ❌ 未実装 |

→ 寄付は残しつつ、サブスクを「本流の課金」として追加する設計にする。

---

## 4月時点での結論

実装は **5月以降**。
4月は設計を固めることを優先し、以下を決定事項とする：

1. 基本機能は永久無料
2. AI 系機能（分析・音声・画像）は月次カウントで制限
3. 有料プランは月額 ¥300〜¥500 の Stripe Subscription
4. 寄付導線は並存させる
