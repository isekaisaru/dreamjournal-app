# 夢の森（プロフィール別 夢の木）設計

作成日: 2026-06-11
ステータス: 設計確定（ブレインストーミング承認済み）
前提: Phase 5 dream_profiles MVP は実装済み（docs/phase5-dream-profiles-design.md 参照）

---

## 1. 目的

プロフィール（自分・家族・ペット）ごとの「夢の木」が並ぶ**夢の森**を作る。

- 夢を書くほど、その人の木が**育つ**（大きく茂る）
- 最近の夢が**光る実**として木になり、タップするとその夢が開く
- 夜の森を眺めて癒される、YumeTree の世界観（家族の記録箱・箱庭）の象徴となる画面

ブレインストーミングで確定した方向性:

| 決定事項 | 選択 |
|---|---|
| コンセプト | 育つ木 ＋ 癒しの実りの木（光る実）の組み合わせ |
| スケール対応 | 最近の実だけタップ可能 ＋ 育ちレベル（夢が何件でも美しい） |
| プロフィール間の行き来 | 森ビュー（俯瞰）→ タップで1本の木へズーム |
| v1の盛り要素 | 昔の夢リスト・育つ/実る演出・保存直後のお祝い |
| 演出方針 | できる限りリッチでモダン（後述） |
| マスコット | モルペウスを案内人として配置（後述） |
| v1でやらないこと | シェアカード・アーカイブ済みプロフィールの木・感情別の枝分かれ |

## 2. 画面と動線

| 画面 | URL | 内容 |
|---|---|---|
| 森ビュー | `/forest` | active な全プロフィールの木が地面に並ぶ。木の大きさ＝夢の総数。木をタップで1本ビューへ |
| 1本の木 | `/forest/[profileId]` | その人の大きな木。最近の実（タップで夢詳細へ）＋育ちレベル＋昔の夢リスト |
| 入口 | ヘッダー | AuthNav に「🌳もり」NavItem を追加（「おうち」の隣、`/forest` へ） |

認証必須。未ログインは `/login` へリダイレクト（既存ページと同じパターン）。

## 3. 木の見た目のルール（純粋関数）

すべて `frontend/lib/forest.ts` の純粋関数として実装し、単体テストで守る。

### 育ちレベル

夢の総数で決まる。

| 総数 | レベル | 表示名 |
|---|---|---|
| 0 | 0 | 芽（地面に芽だけ） |
| 1–4 | 1 | 🌱 苗 |
| 5–14 | 2 | 🌿 若木 |
| 15–29 | 3 | 🌳 木 |
| 30–59 | 4 | 🌳 大樹 |
| 60+ | 5 | 🌲 古木 |

木のサイズ（幹の太さ・茂みの大きさ）はレベルで段階的に大きくなる。上限あり（古木以上は育たない）。

### 光る実

- **最近の夢 12 件**だけを実として表示・タップ可能
- 13 件目より古い夢は「茂みの繁り」として表現（タップ不可、昔の夢リストから開ける）
- 実の色: その夢の感情タグから決める。感情がなければプロフィールカラー
- 実の位置: **夢の ID をシードにした決定的な擬似乱数**で配置する。再描画やリロードで実の位置が変わらないこと（テスト対象）

## 4. データの流れ

バックエンドの新規実装は **`dreams_count` の追加 1 点のみ**。それ以外は既存 API を再利用する。

### 森ビュー

- `GET /dream_profiles` を呼ぶ
- **変更**: `DreamProfilesController#profile_json` のレスポンスに `dreams_count`（そのプロフィールに紐づく夢の総数）を追加する
  - N+1 を避けるため、index では `Dream.where(dream_profile_id: ids).group(:dream_profile_id).count` の 1 クエリでまとめて取得して合成する（counter cache はマイグレーションが必要になるため採用しない）
- 森には active プロフィールのみ表示（archived は表示しない）

### 1本の木

- `GET /dreams?dream_profile_id=:id` を呼ぶ（既存・変更不要）
  - ページングなしで全件、感情タグ込み、重い generated_image_url は除外済み
- 先頭 12 件（created_at 降順）＝光る実、それ以降＝昔の夢リスト

### お祝い

- `DreamForm.tsx` の保存成功時に「実がなった！」演出を表示し、「もりを見にいく」リンクを添える
- API 変更なし

## 5. リッチ＆モダン演出

framer-motion（導入済み）と CSS アニメーションで実装する。

| 演出 | 内容 | 実現方法 |
|---|---|---|
| 星空背景 | またたく星・月のグロー・たまに流れ星 | CSS/SVG アニメ（軽量） |
| パララックス | 奥の丘・中景・手前の地面が視差で動く | scroll/pointer に応じた transform |
| 蛍の光 | 森をふわふわ漂う光の粒 | framer-motion |
| 木のゆらめき | 茂みがゆっくり呼吸するように揺れる | CSS keyframes |
| 実のグロー | 実が脈打つように明滅、タップでポンと弾む | framer-motion |
| 森→木のズーム遷移 | 木をタップすると吸い込まれるように拡大遷移 | framer-motion |
| お祝い | 実がポンと生まれ光が弾ける＋モルペウスが祝福 | framer-motion |

**アクセシビリティ**: `prefers-reduced-motion` を尊重し、設定時は全アニメーションを静的表示に切り替える。実はボタンとして `aria-label`（例: 「◯月◯日の夢: タイトル」）を付ける。

**パフォーマンス**: アニメは transform / opacity のみ（reflow を起こさない）。粒子の数は控えめ（蛍 10 個程度）。

## 6. モルペウスの配置とナビゲーション

新規画像は作らず、既存の `MorpheusImage`（10 variant）を再利用する。
ホームの `MorpheusAssistant` の吹き出しパターン（framer-motion + 閉じるボタン + localStorage で初回判定)を踏襲した `ForestGuide.tsx` を新設する。

| 場所 | 役割 | variant |
|---|---|---|
| 森ビュー右下 | 案内人。初回:「ここは ゆめの もり。きを たっぷしてみて！」。以降は状況に応じた一言（夢 0 の家族がいれば「◯◯のきも そだててあげてね」） | `home` |
| 1本の木ビュー | 育ちレベルの実況（「たろうのきは わかぎに そだったよ！」）。吹き出しに「もりにもどる」「ゆめをかく」ナビボタンを添える | `praise` など |
| 夢 0 本の木（空状態） | 芽だけの地面で「ゆめを かいて きを そだてよう」→ 夢作成画面へ誘導 | `empty` |
| 保存直後のお祝い | 「実がなったよ！」→「もりを見にいく」リンク | `praise` / `reward` |

## 9. テスト方針

- **単体テスト（Vitest/Jest 既存設定に従う）**: `lib/forest.ts` の純粋関数
  - 育ちレベルの境界値（0, 1, 4, 5, 14, 15, 29, 30, 59, 60）
  - 実の位置が同じ夢 ID で常に同じであること（決定性）
  - 実の色（感情あり／なし／プロフィールカラーへのフォールバック)
- **request spec（RSpec）**: `dream_profiles#index` が `dreams_count` を返すこと・N+1 が増えないこと
- **E2E（Playwright）**: 森 → 木 → 実タップ → 夢詳細 の導線 1 本
  - UI コピーを変える場合は同 PR で E2E セレクターも直す（プロジェクト方針）
  - 重複テキストは `locator().filter()` で絞り込む

## 10. エラー・空状態の扱い

| 状況 | 挙動 |
|---|---|
| プロフィール取得失敗 | toast でエラー表示（既存パターン踏襲） |
| 夢 0 件のプロフィール | 芽＋モルペウスの誘導（空状態デザイン） |
| 存在しない/他人の profileId を直打ち | プロフィール一覧に該当なし → 森へリダイレクト |
| アーカイブ済みプロフィールの URL 直打ち | 同上（v1 では木を表示しない） |
